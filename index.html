Git Tracker Removal Guide

Remove-Item -Recurse -Force .git

Explanation:

  • Remove-Item: Deletes the specified item (in this case, the .git directory)
  • -Recurse: Ensures that all files and subdirectories inside .git are also deleted
  • -Force: Overrides any read-only or hidden attributes
  • .git: Specifies the Git tracking folder to be removed

Steps:

0. Verify that the .git directory exists in your cloned repository:

ls -Force

1. Navigate to the folder where your cloned repository is located:

cd path\to\your\cloned\repo

2. Run the command:

Remove-Item -Recurse -Force .git

3. Verify that the .git directory is gone:

ls -Force

Outcome:

The folder is no longer a Git repository and all tracking information has been removed.
